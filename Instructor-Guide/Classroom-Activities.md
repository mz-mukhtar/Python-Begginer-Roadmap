# 🧑‍💻 Classroom Activities

This guide provides instructors with a toolkit of reusable, high-engagement classroom activities designed to make Python lessons interactive, collaborative, and practical.

---

## Warm-Up Activities (5–10 Minutes)

Use these short activities at the start of a session to energize students and review previous concepts:
- **Predict the Output**: Project a 5-line Python code snippet on the screen and ask students to write down what will print before running it.
- **Find the Error**: Display a code snippet containing a subtle syntax or logic bug and challenge students to spot line number and cause.
- **Explain the Code**: Ask a student to articulate in plain spoken English what a specific function or loop accomplishes step-by-step.
- **Review Question**: Ask a conceptual multiple-choice or open question summarizing the previous session's core topic.
- **Code Ordering Activity**: Provide 5 jumbled lines of code on index cards or slides and ask students to arrange them into correct execution order.

---

## Pair Activities

### Pair Programming
Pair programming pairs two learners at one computer to build collaborative coding habits:
- **Driver**: Holds the keyboard and mouse, actively typing the code and focusing on syntax.
- **Navigator**: Observes the screen, reviews logic, spots typos, and thinks ahead about the next implementation step.
- **Rotation Rule**: Instructors should set a timer to switch roles every 15–20 minutes so both partners practice both roles equally.

### Code Explanation Partner
Students swap scripts with a partner and take turns explaining how their partner's code works line-by-line.

### Debugging Partners
Each student writes a small script containing one deliberate bug, then trades computers with their partner to diagnose and fix the issue.

## Group Activities

### Human Algorithm
Students work in small groups of 3–4 to write out unambiguous, step-by-step instructions for a real-world task (e.g., making tea or sorting books). Teams test instructions literally to understand algorithmic precision.

### Data Structure Activity
Give groups a set of real-world scenarios and ask them to categorize which Python data structure fits best:
- **List**: Ordered shopping items that can be added or removed.
- **Tuple**: Fixed GPS coordinates `(latitude, longitude)` that should never change.
- **Dictionary**: Student profile record mapping `"id"` to `"STU-001"`.
- **Set**: Unique list of attended workshop topics without duplicates.

### Project Planning Group
Teams collaborate to brainstorm project features, design interactive CLI menu layouts, and complete the Project Planning worksheet together.

## Coding Activities

- **Complete Missing Code**: Provide a starter file with blank `TODO` blocks that students must fill in to make the program work.
- **Improve Existing Code**: Give students a working script that uses poor variable names and ask them to refactor it for clean readability.
- **Add a Feature**: Challenge students to take a completed script and implement one new menu option or validation check.
- **Refactor a Function**: Take a repetitive 40-line procedural block and guide students to convert it into two reusable functions.
- **Fix Broken Code**: Provide a script with three real-world tracebacks (`ValueError`, `KeyError`, `IndexError`) for students to repair.
- **Test Another Student's Code**: Students run their peer's application and attempt to find edge cases or inputs that crash it.

## Weekly Activity Recommendations

| Week | Recommended Activity | Focus Skill |
| --- | --- | --- |
| **Week 1** | Predict the Output & Find the Error | Variable syntax, type casting, and conditional branching accuracy |
| **Week 2** | Human Algorithm & Data Structure Categorization | Tracing loop execution and choosing the right collection type |
| **Week 3** | Refactor a Function & Pair Programming | Decomposing large scripts into single-responsibility functions |
| **Week 4** | Fix Broken Code & File Path Inspector | Handling string sanitization and reading/writing text/JSON files |
| **Week 5** | Debugging Partners & Traceback Diagnosis | Catching exceptions gracefully and logging diagnostic events |
| **Week 6** | Object Blueprint Modeling & Code Explanation Partner | Designing class attributes/methods and explaining object instances |
| **Week 7** | Package Layout Challenge & Git Collaborative Push | Organizing multi-file packages and pushing clean commits to GitHub |
| **Week 8** | Peer Capstone Testing & Presentation Walkthroughs | Executing automated unit test suites and presenting capstone projects |

## Student Support Activities

- **Beginner Support Group**: Gather students needing extra help for a guided, step-by-step whiteboard walkthrough of foundational logic.
- **Challenge Extension**: Provide advanced learners who finish early with stretch challenge tasks (e.g., adding CSV export or search filtering).
- **Peer Support**: Pair confident students with peers needing assistance to foster collaborative teaching and communication skills.
- **Review Station**: Set up an open Q&A table where students can bring specific tracebacks for guided diagnostic support.

---

## Classroom Rules & Code of Conduct

Establish these foundational rules during Session One to maintain a supportive learning culture:
- **Ask questions freely**: No foundational programming question is ever too simple.
- **Help without giving complete answers**: When assisting a peer, guide them with questions rather than typing code on their keyboard.
- **Respect others**: Celebrate diverse backgrounds and learning speeds across the classroom.
- **Do not copy projects**: Original work builds real developer skills; copying complete projects bypasses learning.
- **Protect personal data**: Always use fictional sample data (`sara@example.com`, `STU-001`) in exercises and projects.
- **Do not share credentials**: Never commit or share private passwords, API keys, or authentication tokens.
