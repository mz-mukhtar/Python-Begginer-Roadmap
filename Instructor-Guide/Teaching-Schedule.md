# 📅 Eight-Week Teaching Schedule

This comprehensive teaching schedule provides instructors with a structured pacing guide for delivering all 24 sessions of the **Python Beginner to Practical Developer Roadmap**.

---

## Complete 24-Session Schedule Overview

| Week | Session | Main Topic | Practical Activity | Expected Outcome |
| --- | --- | --- | --- | --- |
| **Week 1** | Session 1 | Introduction to Python & Syntax | Personal Introduction Script | Students install Python and execute their first script |
| **Week 1** | Session 2 | Variables, Data Types & Operators | Interactive Simple Calculator | Students collect input, cast types, and perform calculations |
| **Week 1** | Session 3 | Conditional Logic (`if`/`elif`/`else`) | Smart Student Grade System | Students build interactive score grading logic |
| **Week 2** | Session 1 | `while` & `for` Loops | Loop Counter & Menu Loop | Students automate repetitive tasks and loop through ranges |
| **Week 2** | Session 2 | Lists and Tuples | Dynamic Classroom Roster | Students index, slice, append, and sort list collections |
| **Week 2** | Session 3 | Dictionaries & Sets | Student Information Collector | Students structure key-value records and remove duplicates |
| **Week 3** | Session 1 | Function Fundamentals (`def`) | Reusable Utility Functions | Students write modular functions with parameters |
| **Week 3** | Session 2 | Return Values & Scope | Calculation Module | Students return values and manage parameter defaults |
| **Week 3** | Session 3 | Program Decomposition | Functional Student Manager | Students refactor CLI applications into clean functions |
| **Week 4** | Session 1 | Strings & Validation Methods | Text Search & Input Validator | Students sanitize and format user string input |
| **Week 2-4** | Session 2 | Text Files & `pathlib` | Permanent Roster File Writer | Students open, read, write, and append local files |
| **Week 4** | Session 3 | JSON, CSV & Persistence | Persistent Student Manager | Students serialize records to JSON and export CSV files |
| **Week 5** | Session 1 | Exceptions & Input Validation | Crash-Proof Numeric Loader | Students catch `ValueError` and handle missing data |
| **Week 5** | Session 2 | Modules & Standard Library | Modular Helper Script | Students import standard utilities and custom Python files |
| **Week 5** | Session 3 | Debugging & Diagnostic Logging | Reliable Student Manager | Students debug tracebacks and configure event loggers |
| **Week 6** | Session 1 | Classes, Objects & `__init__` | Student Class Blueprint | Students instantiate custom objects with instance attributes |
| **Week 6** | Session 2 | Object Collections & Serialization | Object Roster Manager | Students manage lists of objects and convert to dicts |
| **Week 6** | Session 3 | Basic Inheritance & OOP Project | OOP Student Manager | Students create child classes extending parent behaviors |
| **Week 7** | Session 1 | Virtual Environments & `pip` | Package Setup & Requirements | Students isolate project dependencies cleanly |
| **Week 7** | Session 2 | Git & GitHub Version Control | Repository Setup & First Push | Students track commits and push to GitHub |
| **Week 7** | Session 3 | Clean Architecture & Package Layout | Professionally Organized Project | Students organize multi-file packages (`models/`, `services/`) |
| **Week 8** | Session 1 | REST APIs & `requests` | Live Weather/API Inspector | Students make HTTP GET requests and parse JSON payloads |
| **Week 8** | Session 2 | Automated Testing (`unittest`) | Unit Test Verification Suite | Students write test cases and assert expected behavior |
| **Week 8** | Session 3 | Capstone Project & Presentation | Capstone Presentation Showcase | Students finalize, test, and present their capstone project |

---

## Recommended Three-Hour Session Structure

| Time | Activity | Description |
| --- | --- | --- |
| **0–15 minutes** | Review | Quick warm-up questions and review of previous session concepts |
| **15–45 minutes** | Concept introduction | Instructor introduces new core theory with diagrams and real-world analogies |
| **45–90 minutes** | Instructor demonstration | Live coding demonstration building clear examples step-by-step |
| **90–105 minutes** | Break | Rest break to keep students refreshed |
| **105–150 minutes** | Guided practice | Students complete structured exercises (`exercises/starter/`) with instructor support |
| **150–170 minutes** | Independent activity | Students tackle the independent challenge or weekly project feature |
| **170–180 minutes** | Review and assignment | Session wrap-up, conceptual quiz check, and progress checklist update |

> **Note**: Instructors may adjust timing blocks flexibly based on classroom pacing and student support needs.

---

## Weekly Structure

Every week follows a structured pedagogical progression:

```text
Session One: New concepts
     ↓
Session Two: Concept development
     ↓
Session Three: Application and project
```

---

## Complete Week Schedule Breakdown

### Week One — Python Foundations
- **Session One**:
  - **Topic**: Introduction to Python
  - **Preparation**: Ensure Python 3 and VS Code are installed on all student computers.
  - **Activity**: Personal Introduction Program (`print()` statements).
  - **Assessment**: Verify every student can execute a `.py` script from the terminal.
- **Session Two**:
  - **Topic**: Variables, Data Types, Input, and Operators
  - **Preparation**: Review type casting (`int()`, `float()`) and arithmetic operators.
  - **Activity**: Simple Calculator mini-project.
  - **Assessment**: Check that students correctly cast `input()` strings before math calculations.
- **Session Three**:
  - **Topic**: Conditional Logic (`if`/`elif`/`else`)
  - **Preparation**: Review boolean comparisons and logical operators (`and`, `or`, `not`).
  - **Activity**: Smart Student Grade System weekly project.
  - **Assessment**: Evaluate weekly project grading accuracy and boundary validation.

### Week Two — Loops and Python Collections
- **Session One**:
  - **Topic**: `while` and `for` Loops
  - **Preparation**: Prepare loop tracing diagrams showing counter increments.
  - **Activity**: Interactive menu loop and number range generator.
  - **Assessment**: Ensure students avoid infinite loops and use `break`/`continue` properly.
- **Session Two**:
  - **Topic**: Lists and Tuples
  - **Preparation**: Review 0-based indexing and list methods (`append`, `pop`).
  - **Activity**: Dynamic Classroom Roster exercise.
  - **Assessment**: Verify students can modify lists without raising `IndexError`.
- **Session Three**:
  - **Topic**: Dictionaries, Sets, and Weekly Project
  - **Preparation**: Prepare key-value lookup examples and unique set demonstrations.
  - **Activity**: Student Information Collector weekly project.
  - **Assessment**: Check that students structure multi-field student dictionaries correctly.

### Week Three — Functions and Program Organization
- **Session One**:
  - **Topic**: Function Fundamentals (`def`)
  - **Preparation**: Prepare diagrams showing inputs (parameters) and behavior.
  - **Activity**: Reusable greeting and math utility functions.
  - **Assessment**: Confirm students can define and invoke functions with positional arguments.
- **Session Two**:
  - **Topic**: Return Values and Scope
  - **Preparation**: Review `return` statements vs. `print()`, and local vs. global scope.
  - **Activity**: Modular grading calculation functions.
  - **Assessment**: Check that functions return data cleanly rather than printing directly.
- **Session Three**:
  - **Topic**: Program Decomposition
  - **Preparation**: Review single-responsibility principle and menu refactoring.
  - **Activity**: Function-Based Student Management System weekly project.
  - **Assessment**: Evaluate code cleanliness and separation of menu and CRUD logic.

### Week Four — Strings and Data Persistence
- **Session One**:
  - **Topic**: Strings in Depth
  - **Preparation**: Review string slicing, formatting (`f""`), and validation (`isdigit()`).
  - **Activity**: Text sanitizer and ID format validator.
  - **Assessment**: Verify students validate text before numeric conversion.
- **Session Two**:
  - **Topic**: Text Files and `pathlib`
  - **Preparation**: Review context managers (`with open()`) and cross-platform file paths.
  - **Activity**: Permanent Roster File Writer.
  - **Assessment**: Ensure files are read and written without leaving file handles open.
- **Session Three**:
  - **Topic**: JSON, CSV, and Persistent Applications
  - **Preparation**: Review `json.dump()` / `json.load()` and standard `csv.writer()`.
  - **Activity**: Persistent Student Management System weekly project.
  - **Assessment**: Check that student rosters save and reload properly across restarts.

### Week Five — Reliable Python Applications
- **Session One**:
  - **Topic**: Errors and Exception Handling
  - **Preparation**: Review traceback reading and `try`/`except ValueError` blocks.
  - **Activity**: Crash-Proof Numeric Input Loader.
  - **Assessment**: Verify applications handle invalid letter entries without crashing.
- **Session Two**:
  - **Topic**: Modules and the Python Standard Library
  - **Preparation**: Prepare imports from `math`, `datetime`, and custom `.py` helper files.
  - **Activity**: Custom Helper Module integration.
  - **Assessment**: Confirm students can split utility functions into helper modules cleanly.
- **Session Three**:
  - **Topic**: Debugging and Diagnostic Logging
  - **Preparation**: Review VS Code debugging tools and standard `logging.basicConfig()`.
  - **Activity**: Reliable Student Management System weekly project.
  - **Assessment**: Evaluate error resilience and audit file logging (`system.log`).

### Week Six — Object-Oriented Programming
- **Session One**:
  - **Topic**: Classes and Objects
  - **Preparation**: Review OOP diagrams showing class blueprints vs. concrete objects.
  - **Activity**: Student Class Blueprint with `__init__` constructor.
  - **Assessment**: Verify students instantiate objects correctly with `self` attributes.
- **Session Two**:
  - **Topic**: Building Class-Based Applications
  - **Preparation**: Review object lists and `to_dict()` serialization helper methods.
  - **Activity**: Object Roster Manager.
  - **Assessment**: Check that object attributes update properly through instance methods.
- **Session Three**:
  - **Topic**: Basic Inheritance and OOP Project
  - **Preparation**: Review parent/child inheritance (`super().__init__()`).
  - **Activity**: Object-Oriented Student Management System weekly project.
  - **Assessment**: Evaluate class encapsulation and clean separation of responsibilities.

### Week Seven — Python Development Tools
- **Session One**:
  - **Topic**: Virtual Environments and Packages
  - **Preparation**: Review `python -m venv .venv`, environment activation, and `pip`.
  - **Activity**: Virtual environment setup and `requirements.txt` creation.
  - **Assessment**: Confirm students install packages inside their active `.venv`.
- **Session Two**:
  - **Topic**: Git and GitHub
  - **Preparation**: Review `git init`, `status`, `add`, `commit`, `push`, and `.gitignore`.
  - **Activity**: Local Git repository creation and GitHub remote connection.
  - **Assessment**: Verify no `.venv` or cache folders are pushed to GitHub.
- **Session Three**:
  - **Topic**: Project Structure and Clean Code
  - **Preparation**: Review multi-file package layouts (`models/`, `services/`, `utilities/`).
  - **Activity**: Professionally Organized Student Management System weekly project.
  - **Assessment**: Check modular separation and entrypoint clean execution (`main.py`).

### Week Eight — APIs, Testing, and Capstone Development
- **Session One**:
  - **Topic**: Working With APIs
  - **Preparation**: Review HTTP GET requests, status codes (`200`), and JSON response parsing.
  - **Activity**: Live REST API Inspector script.
  - **Assessment**: Verify students handle network requests and inspect status codes safely.
- **Session Two**:
  - **Topic**: Introduction to Automated Testing
  - **Preparation**: Review `unittest.TestCase`, test discovery, and assertions (`assertEqual`).
  - **Activity**: Automated unit test suite creation.
  - **Assessment**: Ensure students can run and pass `python -m unittest discover`.
- **Session Three**:
  - **Topic**: Capstone Project Showcase
  - **Preparation**: Prepare presentation timer, demo projector, and grading rubrics.
  - **Activity**: Final Capstone Project Demonstrations.
  - **Assessment**: Grade capstone applications against the 100-point Project Rubric.

---

## Course Milestones

Monitor class-wide readiness at these five critical checkpoints:

- **End of Week Two**: Students understand core Python syntax, variables, conditionals, loops, and data collections (`list`, `dict`).
- **End of Week Four**: Students can create file-based programs that save and load permanent data (`pathlib`, JSON, CSV).
- **End of Week Six**: Students can architect applications using Object-Oriented Programming principles (`class`, `self`, methods).
- **End of Week Seven**: Students can organize professional multi-file software packages and collaborate using Git/GitHub.
- **End of Week Eight**: Students complete, test, document, and present a complete, production-ready capstone project.
