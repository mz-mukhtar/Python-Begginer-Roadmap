# 🚀 Python Beginner to Practical Developer Roadmap — Projects

Welcome to the **Projects Directory** of the **Python Beginner to Practical Developer Roadmap**!

Projects are where Python theory becomes practical developer skill. By building complete applications, you will learn how to structure code, debug issues, manage data, and solve real-world problems.

---

## 📌 About the Projects

This directory is divided into two main areas:
1. **Weekly Projects (`weekly-projects/`)**: One continuous application—the **Student Management System**—that gradually evolves across the 8 weeks of the course as you learn new Python capabilities.
2. **Final Projects (`final-projects/`)**: Five complete capstone project tracks where you apply everything you have learned across the entire 8-week roadmap.

---

Watch how a single application improves week by week:

| Week | Project Name | Key Python Concepts Applied |
| :--- | :--- | :--- |

| **Week 6** | [Object-Oriented Student Management System](weekly-projects/week-six/) | Classes, Objects, Methods, Constructors, Basic Inheritance |
| **Week 7** | [Professionally Organized Student Management System](weekly-projects/week-seven/) | Multi-File Architecture, Models/Services/Utilities, `.gitignore`, `requirements.txt` |

## 🏆 Final Projects

In Week 8, learners select one of five capstone project tracks to build independently:
1. **[Student Management System](final-projects/student-management-system/)**
2. **[Contact Book](final-projects/contact-book/)**
3. **[Expense Tracker](final-projects/expense-tracker/)**
4. **[Quiz Application](final-projects/quiz-application/)**
5. **[To-Do List Manager](final-projects/todo-list-manager/)**

Each track includes:
- A **`starter/`** directory containing clean project scaffolding, schemas, and `TODO` guidance.
- A **`reference-solution/`** directory containing a complete, runnable, well-documented implementation with automated tests.

## ▶️ Running Projects

To run any weekly or final project:
```bash
cd weekly-projects/week-four/
python main.py
```
*(Note: Week 1, Week 2, and Week 3 single-file projects use specific module names like `python student_grade_system.py`).*

---

## 💡 Project Rules for Students
- **Understand the Code**: Never copy code without understanding what every line does.
- **Experiment and Modify**: Change prompt messages, add new fields, and test how the application behaves.
- **Read Error Messages**: Tracebacks are your friends—read from the bottom up to locate bugs.
- **Use Version Control**: Commit your progress regularly using Git!
