# 🏆 Final Projects Directory — Python Beginner Roadmap

Welcome to the **Final Projects** directory! Here you will choose and build your capstone application to graduate from the **Python Beginner to Practical Developer Roadmap**.

---

## 📌 About Final Projects
Each final project track allows you to integrate everything you learned across the 8-week course:
- Interactive Command-Line Interfaces (CLI)
- Object-Oriented Domain Models (`models/`)
- Persistent JSON and CSV Data Storage (`data/`)
- Exception Handling and Input Validation
- Multi-File Package Architecture (`services/`, `utilities/`)
- Automated Unit Testing (`tests/`)

---

## 🧭 Available Project Tracks
Students select **one** of the following five tracks:
1. **[Student Management System](student-management-system/)** — Advanced academic registry with courses, IDs, and CSV reporting.
2. **[Contact Book](contact-book/)** — Multi-category personal & professional contact directory.
3. **[Expense Tracker](expense-tracker/)** — Financial ledger tracking spending by category with summary reporting.
4. **[Quiz Application](quiz-application/)** — Interactive CLI examination engine with randomized choices and score calculation.
5. **[To-Do List Manager](todo-list-manager/)** — Productivity task tracking with due dates, priority levels, and completion statuses.

---

## 📁 How Each Track is Structured
Inside each track folder, you will find:
```text
track-name/
├── README.md               # Overview of the capstone track & specifications
├── starter/                # Scaffolded directory structure with TODO comments for independent work
└── reference-solution/     # Complete, runnable, tested reference implementation
```

> **Important Learning Rule**: Try building your project inside `starter/` first! Consult `reference-solution/` only as a learning resource or to compare architectural choices after you make your own implementation attempt.
