# 📘 Week 8 – Session One: Working With APIs

## 📌 Session Overview

This session introduces students to REST APIs and HTTP requests. Students will learn how web services communicate using structured JSON data and how to use Python's third-party `requests` library to fetch live internet data into their programs.

## 🎯 Learning Objectives

Students will be able to:
- Explain what an Application Programming Interface (API) is and how client-server communication works
- Identify key HTTP concepts including endpoints, URLs, HTTP methods (`GET`), and status codes (`200 OK`, `404 Not Found`)
- Install and use the Python `requests` library to make HTTP `GET` calls
- Parse JSON API responses into Python dictionaries and lists
- Build a simple interactive utility that fetches live information from a public API

## 📚 Planned Topics

- What an API is
- Requests
- Responses
- URLs
- Endpoints
- HTTP methods
- Status codes
- JSON responses
- Installing and using `requests`
- Building a simple API-based application

## 🛠️ Planned Practical Activity

**Live API Information Fetcher**: Students will build a clean Python script that queries a public JSON API, verifies HTTP status codes, extracts relevant fields, and displays formatted results to the user.
