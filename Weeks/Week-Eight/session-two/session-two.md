# 📘 Week 8 – Session Two: Introduction to Testing

## 📌 Session Overview

This session introduces software testing principles and automated verification. Students will learn the difference between manual testing and automated assertions, write standard `assert` checks, and build introductory unit tests using Python's built-in `unittest` module.

## 🎯 Learning Objectives

Students will be able to:
- Explain why testing prevents bugs and ensures long-term application stability
- Compare expected results against actual results systematically
- Use inline `assert` statements to verify function outputs
- Structure automated test cases using Python's `unittest.TestCase` class
- Run unit test suites and interpret test failure reports

## 📚 Planned Topics

- Why testing is important
- Expected results
- Actual results
- Manual testing
- Assertions
- Automated testing
- Introduction to `unittest`

## 🛠️ Planned Practical Activity

**Automated Test Suite**: Students will write automated unit test cases verifying the calculation and data-handling functions of their student project using `unittest`.
