# 📘 Week 4 – Session Three: JSON, CSV, and Persistent Applications

## 📌 Session Overview

This session teaches students how to make data survive between program executions using industry-standard formats: JSON and CSV. Students will practice reading and writing structured data files and upgrade their project into the **Persistent Student Management System**.

## 🎯 Learning Objectives

Students will be able to:
- Serialize Python dictionaries and lists into structured JSON files
- Load JSON objects and arrays from disk back into Python collections
- Read and write tabular CSV files using headers, rows, and columns
- Choose between JSON and CSV formats based on project data requirements
- Upgrade their student management application with permanent disk storage

## 📚 Planned Topics

- JSON
- JSON objects
- JSON arrays
- Saving JSON
- Loading JSON
- CSV files
- Rows
- Columns
- Headers
- Reading CSV
- Writing CSV
- Persistent Student Management project

## 🛠️ Planned Weekly Project

**Persistent Student Management System**: An upgraded application that loads stored student records from JSON or CSV files upon startup and saves all modifications permanently to disk before exiting.
