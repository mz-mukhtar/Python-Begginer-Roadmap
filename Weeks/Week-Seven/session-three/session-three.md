# 📘 Week 7 – Session Three: Project Structure and Clean Code

## 📌 Session Overview

This session focuses on professional Python project architecture and clean coding practices. Students will learn how to structure multi-file projects cleanly, write readable docstrings, and separate application entry points (`main.py`) from underlying module logic.

## 🎯 Learning Objectives

Students will be able to:
- Organize Python projects into clear folder hierarchies (`main.py`, modules, data folders, test directories)
- Apply descriptive naming conventions for files, functions, and classes
- Write informative module, class, and function docstrings
- Use the `if __name__ == "__main__":` idiom to create clean script entry points
- Eliminate duplicate code and refactor disorganized scripts into structured packages

## 📚 Planned Topics

- Organizing Python projects
- `main.py`
- Modules
- Data folders
- Tests folders
- README files
- Descriptive naming
- Avoiding duplicate code
- Comments
- Docstrings
- `if __name__ == "__main__":`

## 🛠️ Planned Practical Activity

**Project Architecture Refactoring**: Students will reorganize a monolithic single-file application into a clean multi-module folder structure complete with an entry point (`main.py`), separate module files, and comprehensive documentation.
