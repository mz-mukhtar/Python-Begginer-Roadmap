# 📘 Week 7 – Session Two: Git and GitHub

## 📌 Session Overview

This session teaches students version control using Git and collaborative cloud hosting with GitHub. Students will practice tracking file changes, creating commits, and pushing repositories to GitHub safely.

## 🎯 Learning Objectives

Students will be able to:
- Explain why version control is critical for modern software development
- Initialize and inspect local Git repositories (`git init`, `git status`)
- Stage and commit code changes with descriptive messages (`git add`, `git commit`)
- Push local projects to GitHub and clone remote repositories (`git push`, `git clone`)
- Configure a `.gitignore` file to exclude virtual environments and temporary files

## 📚 Planned Topics

- Version control
- Git
- GitHub
- Repositories
- `git init`
- `git status`
- `git add`
- `git commit`
- `git push`
- Cloning repositories
- `.gitignore`

## 🛠️ Planned Practical Activity

**Repository Publishing**: Students will initialize a local Git repository for their student project, write a `.gitignore` file, make initial commits, and push their codebase to a remote GitHub repository.
