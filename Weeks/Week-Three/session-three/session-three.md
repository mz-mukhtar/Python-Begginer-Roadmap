# 📘 Week 3 – Session Three: Program Decomposition and Functional Project

## 📌 Session Overview

This session focuses on clean program design and modular architecture. Students will learn how to break complex procedural scripts into focused, single-responsibility functions, build clean CLI menus, eliminate duplicate code, and architect the **Functional Student Management System**.

## 🎯 Learning Objectives

Students will be able to:
- Decompose monolithic programs into small, modular functions
- Apply the Single Responsibility Principle to function design
- Eliminate duplicate code patterns through parameterized functions
- Architect an interactive command-line menu driven by function calls
- Assemble and test the **Functional Student Management System** weekly project

## 📚 Planned Topics

- Breaking large programs into smaller functions
- One clear responsibility per function
- Avoiding duplicate code
- Creating application menus
- Functional Student Management System
- Weekly challenge

## 🛠️ Planned Weekly Project

**Functional Student Management System**: A modular command-line application organized into dedicated functions for adding, viewing, searching, and updating student records using clean menus and reusable helper utilities.
